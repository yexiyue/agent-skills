# 知识主题 → 看板娘人设映射

让卡片"既统一又贴题"的关键,是看板娘的人设要呼应内容主题——讲编程配科技少女,讲中医配汉服药童。本文件给方法和示例库。

## 设计方法(四个维度)

拿到一个知识主题,从四个维度推导人设。**目标不是堆砌细节,而是选出几个能一眼传达"这是讲 X 的"的强符号**:

1. **职业 / 身份符号**:把主题"拟人"成一个职业或身份。编程→程序员 / 机娘;中医→药铺学徒;天文→星之观测者;法律→书记官。
2. **服装 / 配饰**:用 1-2 个标志性单品承载主题。卫衣+耳机、汉服+药杵、披风+星图、西装+卷宗。配饰是最省字又最点题的符号。
3. **配色基调**:主题情绪 → 色板(给十六进制)。科技=冷蓝紫;养生=暖棕米;金融=沉稳藏蓝;儿童=糖果亮色;文史=宣纸褐金。
4. **画风**:匹配受众与调性。严肃专业→日系赛璐珞 / 厚涂;轻松儿童→chibi Q版;文艺→吉卜力 / 水彩;国风→工笔 / 浮世绘混搭。

> 一个好人设 = 1 个身份符号 + 1~2 个标志单品 + 1 套配色 + 1 种画风。再多就稀释记忆点了。

### 强符号优先:主题自带官方 mascot / 品牌色就用它

很多主题本身就有公认的吉祥物或品牌色——**优先采用,贴题度和辨识度立刻拉满**,而不是套泛泛的"科技蓝少女":

- Rust → 锈橙 `#CE422B` + Ferris 螃蟹;Go → 蓝绿 + Gopher;Python → 蓝黄 + 蟒蛇;Docker → 鲸鱼蓝;
- 某品牌 / 产品科普 → 用其官方主色与 logo 元素;
- 某游戏 / 动漫领域 → 用其标志性配色。

把这种强符号做成看板娘的发色 / 印花 / 随身吉祥物 / 道具隐喻(如把 Rust 的 ownership 具象成"所有权钥匙"挂坠),既点题又能贯穿整套卡的副标题与比喻。**没有现成强符号时**,才回退到下面的通用映射表。

## 一个重要区分:系列内固定 vs 跨系列复用

- **系列内**:同一套卡里,人设必须**完全固定**(这是"统一"的来源)。
- **跨系列**:用户可以选择
  - (A) **每个主题换一个贴题人设**——讲什么像什么,代价是没有稳定 IP;
  - (B) **一个固定看板娘 IP 贯穿所有主题**——靠换服装 / 道具点题,但脸 / 发 / 气质不变,长期积累 IP 辨识度(很多账号选这条)。

设计时先问清用户要 A 还是 B。选 B 时,人设的"脸 / 发 / 瞳 / 体型"写进角色档案永久固定,只让"服装 / 道具 / 配色"随主题微调。

## 落地到角色档案

推导出人设后,把它填进 [../assets/mascot-profile-template.md](../assets/mascot-profile-template.md) 的对应字段(外观锁定项、画风、配色),再据此写"角色定稿提示词"。从这一步起,人设就被固化成可复用的常量了。
