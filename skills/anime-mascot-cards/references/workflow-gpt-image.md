# 在 gpt-image 里逐步生图(主打路线)

本 skill 产出的是提示词;真正生图在 gpt-image 完成。这里说明怎么用这些提示词,以及避坑。

## 两种入口

### A. ChatGPT 网页 / App(最简单,推荐普通用户)

GPT-4o 的原生生图(底层即 gpt-image)。流程就是"两步法":

1. **先定稿角色**:新开一个对话,粘贴"角色定稿提示词",生成一张满意的看板娘立绘。不满意就让它微调("换成银色双马尾""背景换纯色"),直到定稿。**这张图就是参考图(锚点)。**
2. **逐张出卡**:对每个知识点,
   - 把第 1 步定稿的立绘**作为图片上传**到对话,
   - 紧接着粘贴该卡的"成卡提示词",
   - 生成。
3. 第 2 张起,可以同时上传"定稿立绘"和"上一张卡",并在提示词里说"风格、版式与上一张完全一致"——双参考能把统一性拉得更高。

> 关键:**每张卡都要带上参考图**。只粘文字不传图 = 退回纯文字生成 = 必漂。

### B. `gpt-image-1` API(适合批量 / 自动化)

用 OpenAI 图像 API。要点:

- **生成参考图**:`images.generate`,`size="1024x1536"`,prompt = 角色定稿提示词。保存这张图。
- **以图带图出卡**:用 `images.edit`(图像编辑 / 参考),把参考图作为输入 `image`,prompt = 成卡提示词。逐个知识点循环调用。
- `size` 只能取 `1024x1024` / `1536x1024` / `1024x1536` / `auto`;知识卡用 `1024x1536`。
- `quality` 可选 `low/medium/high`,知识卡建议 `high`。
- 批量时,**输入图、模板块、约束句保持不变,只换内容字段**,循环即可。

> 注:本 skill 不替用户写调用脚本(保持纯提示词、跨 agent 通用)。如果用户明确要脚本,再单独按其语言 / SDK 写。

## 与 Nano Banana / 即梦的差异(同一套提示词怎么改)

同一套"角色块 + 模板块 + 内容字段"基本通用,**只需调三处写法**:

| 维度 | gpt-image(主打) | Nano Banana Pro | 即梦 / 其它 |
|---|---|---|---|
| 画幅写法 | `1024x1536`(竖) | 可写 `9:16` | 多数支持 `9:16` |
| 参考图 | 上传 1 张;可同时传"立绘 + 上一张" | 上传 ≤3 张最佳,用"图1/图2"指代 | 看平台,一般支持垫图 |
| 一致性措辞 | 自然语言"与参考图 100% 一致" | 同左,且支持"保留图1人物、替换图2文字" | 同左 |
| 中文文字 | 尚可,需校对 | 较强,仍需校对 | 参差,务必校对 |
| 不要写 | `--seed`/`--ar`/`--cref`(MJ 专用) | 同左 | 同左 |

切换平台时:**改的是"尺寸写法 / 参考图指代 / 约束措辞",不变的是"哪些信息必须固定"**——后者由 [consistency-factors.md](consistency-factors.md) 的 11 条决定,本身工具无关。

## 避坑清单

- **中文必校对**:gpt-image 的中文标注常出错别字 / 串字 / 漏字。**逐张人工核对文字**;文字越少越不容易错——每节要点尽量短(给个量化标准,如"每条不超过 15 字")。
- **别靠重抽整图改细节**:想改某张卡的一个图标 / 裙长,用"在这张图基础上,只把 X 改成 Y,其余完全不变"的局部编辑,而不是重新生成——否则容易把脸和画风一起改飞。
- **参考图别丢**:整个系列都用同一张定稿立绘作锚点。换了参考图 = 换了脸。
- **一次别塞太多文字**:版式越复杂、文字越多,出错概率越高。复杂卡可拆成"少文字图 + 后期排版",或多生成几张挑一张。
- **模糊词要定义死**:"简洁""可爱""面向小白"这类词,在提示词里给出可操作的定义(如"每节≤15字、只留 3 个要点"),否则多次生成口径会漂。
- **首图质量决定一切**:角色定稿立绘要舍得花时间打磨——它是整套卡的地基,地基歪了后面全歪。
