# 风格统一的关键因素(本 skill 的灵魂)

一套看板娘知识卡片"看起来是一套",从来不是靠抽卡运气,而是靠下面这些杠杆叠加。按重要性排列,每项都说明**为什么有效**——理解了"为什么",你就能在任何主题、任何工具上复刻,而不是死记某一条模板。

> 主打 gpt-image,所以每条都附"在 gpt-image 上怎么落地"。Nano Banana / MJ 的差异在文末单列。

## 1. 固定角色参考图作锚点(第一杠杆)

先把看板娘定稿成一张高质量立绘,之后**每张卡都把这张图作为输入**(参考图)。

- **为什么有效**:模型有了明确的"身份基准",一致性来自"看着同一张脸画",而不是每次靠文字描述去重新想象一个人。这是所有人物一致性玩法的第一步,也是"统一一套"成立的前提。
- **gpt-image 落地**:在 ChatGPT 里"上传这张立绘 + 写提示词";或 `gpt-image-1` 的图像编辑 / 多图参考接口里把立绘作为输入图。
- **常见错误**:每张卡都从纯文字重新生成角色 → 必然漂。一定要"先定稿一张,再以图带图"。

## 2. 把"不变的"全部固化进固定提示词块

角色、画风、版式、配色、比例,统统写进逐字不动的"角色块 + 模板块"。

- **为什么有效**:把一套卡的"风格 DNA"沉淀成可复制的文本常量。常量不变,风格就不漂。
- **gpt-image 落地**:角色块 + 模板块在每张卡的提示词里**逐字相同**,粘贴时不要顺手改字。

## 3. 每张卡只替换"知识内容字段"

把"会变的内容(标题 + 要点)"和"不能变的风格"物理隔离,每张卡只动内容字段。

- **为什么有效**:变量越少,跨图差异越小。你只在一个受控的小窗口里改东西,其余全是常量。
- **gpt-image 落地**:提示词最底部留一段"知识内容",只改这一段。

## 4. 末尾加强一致性约束句(钉死身份 / 锁住其余)

明确写死:"与参考图中角色 100% 一致——面部、发型、发色、服装、配色、画风完全相同,仅更换文字内容"。第 2 张起再加"风格 / 版式 / 配色与上一张完全一致"。

- **为什么有效**:把"不变"作为硬约束逐字告诉模型,它在生成 / 编辑时会主动 match 参考图的特征、光照、版式,显著压低漂移。
- **gpt-image 落地**:gpt-image 对自然语言约束敏感,这句话很关键。**不要**写 `--seed` / `--cref`(那是 MJ 的,gpt-image 不认)。

## 5. 统一画风关键词("风格指纹")

整套卡复用同一串画风词,如"高质量 2D 插画 / 线条干净利落"、"chibi Q版"、"日系赛璐珞上色"、"厚涂插画"、"吉卜力风"。

- **为什么有效**:画风词是跨图复用的"风格指纹",词不变则风格不漂。混用画风词是整套卡崩坏的常见原因。
- **gpt-image 落地**:把画风词放进模板块,逐字复用。

## 6. 统一版式 / 分区(同一套"信息骨架")

固定"标题区 + 正文 2–4 节 + 看板娘位置"的分区版式,每张卡照搬。

- **为什么有效**:版式固定 → 信息排布一致 → 一眼认出是同一系列。版式是肉眼第二敏感的"成套信号"(仅次于角色脸)。
- **gpt-image 落地**:在模板块里把分区描述写清楚(顶部标题、中部 N 节要点、某侧放看板娘半身)。

## 7. 固定配色(给十六进制,别给"暖色调")

指定一套配色贯穿全套,例如背景 `#F3F0E9`、主色 `#C0392B`、辅色若干。

- **为什么有效**:配色是肉眼**最先**感知的统一信号。给十六进制比给"暖色调"可控得多——"暖色调"每次都会漂成不同的暖色。
- **gpt-image 落地**:在模板块里列出色值;角色块里写"角色配色精确遵循参考图"。

## 8. 统一画幅比例

整套卡固定同一画幅。

- **为什么有效**:比例一变,构图和排版被迫重排,系列感立刻破裂。
- **gpt-image 落地**:知识卡用竖版 `1024x1536`(2:3)。gpt-image 只支持 `1024x1024 / 1536x1024 / 1024x1536 / auto`,**不要写 9:16**(那是 Nano Banana / MJ 的比例,gpt-image 不支持任意比例)。

## 9. 统一收尾词(锁定输出规格)

每张卡共用一段固定收尾词,如"中文标注、文字清晰可读、画面干净专业、留白充足、高清"。

- **为什么有效**:把画质和输出规格也固定下来,减少张与张之间的清晰度 / 排版波动。

## 10. 统一标注 / 引线系统(进阶)

跨卡固定同一个角落签名牌、同一套引线标注层级(主标签 / 子标签)。

- **为什么有效**:标注系统本身是视觉语言的一部分,统一它能让"成套感"更专业,而不只是"角色一样"。

## 11. 先定大纲 / 模板再逐张生成(工作流层)

像做 PPT 一样:先写整套大纲(每张卡讲哪个知识点)→ 再逐张套同一模板生成。

- **为什么有效**:先有统一模板再分发,比一张张随性生成可控得多;也避免"做到第 5 张发现风格已经跑偏"。

---

## 一句话总诀

> **"锚定一张角色图 + 把不变的全锁进固定提示词块 + 每张只换内容字段 + 末尾硬约束句"**——这就是"统一一整套"的全部秘密。其余都是细节。

## 不同工具的一致性手段差异

| 工具 | 一致性主要靠 | 注意 |
|---|---|---|
| **gpt-image / GPT-4o(主打)** | 上传参考图 + 自然语言"保持一致"约束 | 不用 seed;尺寸固定档位;中文文字尚可但需校对 |
| **Nano Banana Pro** | 上传参考图(≤3 张最佳)+ "图1/图2"指定保留/替换 + 约束句 | 一致性 + 中文文字最强;输出带 SynthID 水印;可写 9:16 |
| **Midjourney** | `--seed` 复现 + 垫图 `--iw 2` + 新版 `--cref` | 中文文字弱,不适合文字多的知识卡;靠参数而非自然语言 |

把同一套"角色块 + 模板块 + 内容字段"喂给不同工具时:**改的是"约束语法 / 尺寸写法",不变的是"哪些信息要固定"**——也就是上面 11 条本身是工具无关的。
